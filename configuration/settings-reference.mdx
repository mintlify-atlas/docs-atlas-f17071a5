This page documents all available settings in the `[scrapyd]` section of the configuration file.

## Application Settings

### application

The function that returns the Twisted Application to use.

- **Default:** `scrapyd.app.application`
- **Type:** Python callable

```ini
application = scrapyd.app.application
```

Override this setting if you need to fully control how Scrapyd works. The application must return a Twisted [Application](https://docs.twisted.org/en/stable/core/howto/application.html) object.

### bind_address

The IP address on which the web UI and API listen for connections.

- **Default:** `127.0.0.1`
- **Type:** IP address string
- **Environment variable:** `SCRAPYD_BIND_ADDRESS`

```ini
bind_address = 127.0.0.1
```

<Warning>
  The default `127.0.0.1` only accepts local connections. Use `0.0.0.0` to accept connections from any IPv4 address, or `::0` for both IPv4 and IPv6.
</Warning>

Common values:
- `127.0.0.1` - Listen for local IPv4 connections only (default, most secure)
- `0.0.0.0` - Listen for all IPv4 connections
- `::0` - Listen for all IPv4 and IPv6 connections

<Note>
  If `sysctl` sets `net.ipv6.bindv6only` to `true` (default is `false`), then `::0` listens for IPv6 connections only.
</Note>

To disable the TCP server when using [unix_socket_path](#unix_socket_path), set `bind_address` to empty:

```ini
bind_address =
```

### http_port

The TCP port on which the web UI and API listen for connections.

- **Default:** `6800`
- **Type:** Integer
- **Environment variable:** `SCRAPYD_HTTP_PORT`

```ini
http_port = 6800
```

### unix_socket_path

<Note>
  Added in version 1.5.0
</Note>

The filesystem path of the Unix socket on which the web UI and API listen for connections.

- **Default:** `""` (empty, disabled)
- **Type:** File path
- **Environment variable:** `SCRAPYD_UNIX_SOCKET_PATH`

```ini
unix_socket_path = /var/run/scrapyd/web.socket
```

The file's mode is set to `660` (owner and group, read and write) to control access to Scrapyd.

<Warning>
  If both `bind_address`/`http_port` and `unix_socket_path` are set, both a TCP server and a Unix socket server will start. To use only the Unix socket, set `bind_address` to empty.
</Warning>

### username

<Note>
  Added in version 1.3.0
</Note>

Username for HTTP basic authentication.

- **Default:** `""` (empty, disabled)
- **Type:** String
- **Environment variable:** `SCRAPYD_USERNAME`

```ini
username = admin
```

Both `username` and `password` must be set to non-empty values to enable authentication. See [Authentication](/configuration/authentication) for details.

### password

<Note>
  Added in version 1.3.0
</Note>

Password for HTTP basic authentication.

- **Default:** `""` (empty, disabled)
- **Type:** String
- **Environment variable:** `SCRAPYD_PASSWORD`

```ini
password = secretpassword
```

Both `username` and `password` must be set to non-empty values to enable authentication. See [Authentication](/configuration/authentication) for details.

### spiderqueue

<Note>
  Added in version 1.4.2
</Note>

The class that stores pending jobs.

- **Default:** `scrapyd.spiderqueue.SqliteSpiderQueue`
- **Type:** Python class path

```ini
spiderqueue = scrapyd.spiderqueue.SqliteSpiderQueue
```

The default implementation stores spider queues in SQLite databases named after each project, in the [dbs_dir](#dbs_dir) directory.

You can implement your own spider queue using the `ISpiderQueue` interface.

## Poller Settings

### poller

<Note>
  Added in version 1.5.0
</Note>

The class that tracks capacity for new jobs and starts jobs when ready.

- **Default:** `scrapyd.poller.QueuePoller`
- **Type:** Python class path

```ini
poller = scrapyd.poller.QueuePoller
```

The default `QueuePoller` implementation:
- Adds `max_proc` capacity at startup
- Adds one capacity each time a Scrapy process ends
- Starts jobs every `poll_interval` seconds if there's available capacity

You can implement your own poller using the `IPoller` interface.

### poll_interval

The number of seconds between capacity checks.

- **Default:** `5.0`
- **Type:** Float

```ini
poll_interval = 5.0
```

The poller checks for pending jobs and available capacity every `poll_interval` seconds.

## Launcher Settings

### launcher

The class that starts Scrapy processes.

- **Default:** `scrapyd.launcher.Launcher`
- **Type:** Python class path (Twisted Service)

```ini
launcher = scrapyd.launcher.Launcher
```

Override this to customize how Scrapy processes are launched. The launcher must be a Twisted [Service](https://docs.twisted.org/en/stable/api/twisted.application.service.Service.html).

### max_proc

The maximum number of Scrapy processes to run concurrently.

- **Default:** `0`
- **Type:** Integer (non-negative)

```ini
max_proc = 0
```

When set to `0`, Scrapyd calculates the maximum as:

```
max_proc = max_proc_per_cpu × number_of_CPUs
```

Set to a positive integer to specify an exact limit regardless of CPU count.

### max_proc_per_cpu

The number of concurrent Scrapy processes to run per CPU core.

- **Default:** `4`
- **Type:** Integer (positive)

```ini
max_proc_per_cpu = 4
```

This setting is only used when [max_proc](#max_proc) is set to `0`.

<Tip>
  The optimal value depends on your spiders' resource usage. Web scraping is typically I/O-bound, so you can often run many processes per CPU.
</Tip>

### logs_dir

The directory in which to write Scrapy logs.

- **Default:** `logs`
- **Type:** Directory path

```ini
logs_dir = logs
```

Log files are written to: `{logs_dir}/{project}/{spider}/{job}.log`

To disable log storage, set this to empty:

```ini
logs_dir =
```

<Warning>
  Each `*_dir` setting must point to a different directory.
</Warning>

### items_dir

The directory in which to write Scrapy items.

- **Default:** `""` (empty, disabled)
- **Type:** Directory path

```ini
items_dir = items
```

Item feeds are written to: `{items_dir}/{project}/{spider}/{job}.jl`

When enabled, this setting automatically configures Scrapy's `FEEDS` setting to write items as JSON lines:

```json
{"file:///path/to/items_dir/project/spider/job.jl": {"format": "jsonlines"}}
```

<Note>
  It's recommended to configure feed exports directly in your Scrapy project using the `FEEDS` setting, which supports various storage backends (S3, FTP, etc.) and formats.
</Note>

<Warning>
  Each `*_dir` setting must point to a different directory.
</Warning>

### jobs_dir

<Note>
  Added in version 1.6.0
</Note>

The directory in which to persist Scrapy requests.

- **Default:** `""` (empty, disabled)
- **Type:** Directory path

```ini
jobs_dir = jobs
```

Requests are persisted to: `{jobs_dir}/{project}/{spider}/{job}/`

This setting enables [job persistence](https://doc.scrapy.org/en/latest/topics/jobs.html) by setting Scrapy's `JOBDIR` setting, which allows you to pause and resume spider runs.

<Tip>
  Enable this setting to reduce memory usage for spiders with large request queues.
</Tip>

<Warning>
  Each `*_dir` setting must point to a different directory.
</Warning>

### jobs_to_keep

The number of finished jobs per spider for which to keep log files and item feeds.

- **Default:** `5`
- **Type:** Integer (non-negative)

```ini
jobs_to_keep = 5
```

Scrapyd automatically deletes old log files and item feeds, keeping only the most recent `jobs_to_keep` files per spider.

<Warning>
  Scrapyd deletes old files in the `logs_dir` and `items_dir` directories regardless of origin. Don't store other files in these directories.
</Warning>

To effectively disable cleanup, set this to a very large value:

```ini
jobs_to_keep = 9223372036854775807
```

### runner

The Python script to run Scrapy's CLI.

- **Default:** `scrapyd.runner`
- **Type:** Python module path

```ini
runner = scrapyd.runner
```

Override this to fully control how the Scrapy CLI is invoked. The runner must be a Python [script](https://docs.python.org/3/tutorial/modules.html#executing-modules-as-scripts).

## Web UI and API Settings

### webroot

<Note>
  Added in version 1.2.0
</Note>

The class that defines the web UI and API as a Twisted Resource.

- **Default:** `scrapyd.website.Root`
- **Type:** Python class path

```ini
webroot = scrapyd.website.Root
```

Override this to fully control how the web UI and API work. The webroot must be a Twisted [Resource](https://docs.twisted.org/en/stable/web/howto/using-twistedweb.html#resource-objects).

### prefix_header

<Note>
  Added in version 1.4.2
</Note>

The HTTP header containing the base path of the original request.

- **Default:** `x-forwarded-prefix`
- **Type:** String (header name)

```ini
prefix_header = x-forwarded-prefix
```

This setting is only relevant when Scrapyd runs behind a reverse proxy with a base path (e.g., `/scrapyd`).

The base path must have a leading slash and no trailing slash: `/base/path`

### node_name

<Note>
  Added in version 1.1.0
</Note>

The node name that appears in API responses.

- **Default:** Result of `socket.gethostname()`
- **Type:** String

```ini
node_name = scrapy-server-01
```

### debug

Whether debug mode is enabled.

- **Default:** `off`
- **Type:** Boolean (`on` or `off`)

```ini
debug = off
```

When enabled, API errors return a Python traceback as plain text instead of a generic error message.

<Warning>
  Debug mode should only be enabled in development environments as it may expose sensitive information.
</Warning>

## Storage Settings

### eggstorage

<Note>
  Added in version 1.3.0
</Note>

The class that stores project eggs.

- **Default:** `scrapyd.eggstorage.FilesystemEggStorage`
- **Type:** Python class path

```ini
eggstorage = scrapyd.eggstorage.FilesystemEggStorage
```

The default implementation writes eggs to the [eggs_dir](#eggs_dir) directory.

<Note>
  Eggs are named after the version, replacing characters other than `A-Za-z0-9_-` with underscores. If you frequently use non-word, non-hyphen characters in version names, eggs for different versions may collide.
</Note>

You can implement your own egg storage using the `IEggStorage` interface (for example, to store eggs remotely).

### eggs_dir

The directory in which to write project eggs.

- **Default:** `eggs`
- **Type:** Directory path

```ini
eggs_dir = eggs
```

<Warning>
  Each `*_dir` setting must point to a different directory.
</Warning>

### jobstorage

<Note>
  Added in version 1.3.0
</Note>

The class that stores finished jobs.

- **Default:** `scrapyd.jobstorage.MemoryJobStorage`
- **Type:** Python class path

```ini
jobstorage = scrapyd.jobstorage.MemoryJobStorage
```

Available implementations:

- `scrapyd.jobstorage.MemoryJobStorage` - Stores jobs in memory (lost on restart)
- `scrapyd.jobstorage.SqliteJobStorage` - Stores jobs in a SQLite database (`jobs.db` in [dbs_dir](#dbs_dir))

```ini
# Use SQLite to persist finished jobs across restarts
jobstorage = scrapyd.jobstorage.SqliteJobStorage
```

You can implement your own job storage using the `IJobStorage` interface.

### finished_to_keep

The number of finished jobs for which to keep metadata.

- **Default:** `100`
- **Type:** Integer (non-negative)

```ini
finished_to_keep = 100
```

Finished job metadata is displayed in the web UI and returned by the `listjobs.json` API endpoint.

### dbs_dir

The directory in which to write SQLite databases.

- **Default:** `dbs`
- **Type:** Directory path or `:memory:`

```ini
dbs_dir = dbs
```

This directory is used by:
- `SqliteSpiderQueue` ([spiderqueue](#spiderqueue))
- `SqliteJobStorage` ([jobstorage](#jobstorage))

You can set this to `:memory:` to use in-memory SQLite databases:

```ini
dbs_dir = :memory:
```

<Warning>
  Each `*_dir` setting must point to a different directory.
</Warning>

## Services Section

The `[services]` section defines API endpoints and their implementations.

### Default Services

The default configuration includes these endpoints:

```ini
[services]
schedule.json     = scrapyd.webservice.Schedule
cancel.json       = scrapyd.webservice.Cancel
status.json       = scrapyd.webservice.Status
addversion.json   = scrapyd.webservice.AddVersion
listprojects.json = scrapyd.webservice.ListProjects
listversions.json = scrapyd.webservice.ListVersions
listspiders.json  = scrapyd.webservice.ListSpiders
delproject.json   = scrapyd.webservice.DeleteProject
delversion.json   = scrapyd.webservice.DeleteVersion
listjobs.json     = scrapyd.webservice.ListJobs
daemonstatus.json = scrapyd.webservice.DaemonStatus
```

### Adding Custom Services

To add a custom API endpoint:

```ini
[services]
mywebservice.json = mymodule.MyWebService
```

See the [webservice.py source code](https://github.com/scrapy/scrapyd/blob/master/scrapyd/webservice.py) for implementation examples.

### Removing Services

To disable a default endpoint, set it to empty:

```ini
[services]
daemonstatus.json =
```
