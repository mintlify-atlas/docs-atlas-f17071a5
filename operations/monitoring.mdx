## Overview

Scrapyd provides multiple ways to monitor your scraping operations:
- A web-based user interface
- Log files for debugging
- Item output files
- Status API endpoints

## Web UI

### Accessing the UI

Scrapyd's web UI is available at `http://localhost:6800` by default (or your configured `bind_address` and `http_port`).

The home page displays:
- Links to Jobs, Items, and Logs
- List of deployed projects
- Quick reference for scheduling jobs

From `source/scrapyd/website.py:206-241`, the UI includes:

```html
<ul>
    <li><a href="/jobs">Jobs</a></li>
    <li><a href="/items/">Items</a></li>
    <li><a href="/logs/">Logs</a></li>
</ul>
```

### Jobs Page

The Jobs page (`http://localhost:6800/jobs`) shows three categories:

<Tabs>
  <Tab title="Pending">
    Jobs waiting in the queue to be executed. Shows:
    - Project name
    - Spider name
    - Job ID
    - Cancel button (if enabled)
  </Tab>
  
  <Tab title="Running">
    Currently executing jobs. Shows:
    - Project name
    - Spider name
    - Job ID
    - Process ID (PID)
    - Start time
    - Runtime duration
    - Log link
    - Items link (if items_dir is local)
    - Cancel button (if enabled)
  </Tab>
  
  <Tab title="Finished">
    Completed jobs. Shows:
    - Project name
    - Spider name
    - Job ID
    - Start time
    - Runtime duration
    - Finish time
    - Log link
    - Items link (if items_dir is local)
  </Tab>
</Tabs>

The Jobs page auto-refreshes and provides direct links to log files and scraped items.

### Directory Listings

Scrapyd provides enhanced directory listings with custom styling and a "Last modified" column:

#### Logs Directory

Browse logs at `http://localhost:6800/logs/`. The directory structure is:

```
logs/
├── project1/
│   ├── spider1/
│   │   ├── job1.log
│   │   └── job2.log
│   └── spider2/
│       └── job3.log
└── project2/
    └── spider1/
        └── job4.log
```

#### Items Directory

Browse items at `http://localhost:6800/items/` (if `items_dir` is configured and local). Structure:

```
items/
├── project1/
│   ├── spider1/
│   │   ├── job1.jl
│   │   └── job2.jl
│   └── spider2/
│       └── job3.jl
```

<Note>
  From `source/scrapyd/website.py:91`, Scrapyd adds a "Last modified" column showing timestamps in `YYYY-MM-DD HH:MM` format.
</Note>

## Log Files

### Log File Location

Log files are stored at:

```
{logs_dir}/{project}/{spider}/{jobid}.log
```

The default `logs_dir` is `logs/` (configurable in `scrapyd.conf`).

### Accessing Logs

You can access log files through:

1. **Web browser**: Navigate to `http://localhost:6800/logs/` and browse directories
2. **Direct URL**: `http://localhost:6800/logs/{project}/{spider}/{jobid}.log`
3. **API**: Use `listjobs.json` to get `log_url` for each job
4. **File system**: Directly read files from the logs directory

### Log File Format

Logs contain standard Scrapy output:

```
2024-03-05 10:14:03 [scrapy.core.engine] INFO: Spider opened
2024-03-05 10:14:03 [scrapy.extensions.logstats] INFO: Crawled 0 pages (at 0 pages/min)
2024-03-05 10:14:05 [scrapy.core.engine] INFO: Crawled 1 pages (at 1 pages/min)
2024-03-05 10:14:08 [scrapy.core.scraper] DEBUG: Scraped from <200 http://example.com>
2024-03-05 10:14:10 [scrapy.core.engine] INFO: Closing spider (finished)
```

### When Logs Are Not Available

From `source/docs/api.rst:256-259`:

> `log_url` is `null` in the response if `logs_dir` is disabled or the file doesn't exist.

To disable logging, set `logs_dir` to empty in configuration:

```ini
[scrapyd]
logs_dir =
```

## Item Files

### Item File Location

Item files are stored at:

```
{items_dir}/{project}/{spider}/{jobid}.jl
```

<Warning>
  By default, `items_dir` is empty (disabled). You must configure it to enable item storage.
</Warning>

### Configuring Item Storage

In `scrapyd.conf`:

```ini
[scrapyd]
items_dir = items
```

Your Scrapy project must use a feed exporter to write items. Add to your spider or settings:

```python
custom_settings = {
    'FEEDS': {
        'items/%(name)s/%(time)s.jl': {
            'format': 'jsonlines',
        },
    },
}
```

### Accessing Items

1. **Web browser**: `http://localhost:6800/items/`
2. **Direct URL**: `http://localhost:6800/items/{project}/{spider}/{jobid}.jl`
3. **API**: Use `listjobs.json` to get `items_url`

### Remote Item Storage

From `source/scrapyd/website.py:141`, Scrapyd only provides web access to local item files:

```python
self.local_items = local_items(self.items_dir, urlsplit(self.items_dir))
if self.local_items:
    self.putChild(b"items", File(self.items_dir, "text/plain"))
```

If `items_dir` is a remote URL (e.g., `s3://bucket/items`), the Items link won't appear in the UI, but items are still stored remotely.

## Status API Endpoints

### daemonstatus.json

Get overall daemon status:

```bash
curl http://localhost:6800/daemonstatus.json
```

```json
{
  "node_name": "mynodename",
  "status": "ok",
  "pending": 2,
  "running": 3,
  "finished": 15
}
```

From `source/scrapyd/webservice.py:176-181`:

<ParamField path="pending" type="integer">
  Total number of pending jobs across all projects
</ParamField>

<ParamField path="running" type="integer">
  Number of currently running processes
</ParamField>

<ParamField path="finished" type="integer">
  Number of finished jobs in storage (limited by `finished_to_keep`)
</ParamField>

### listjobs.json

Get detailed information about all jobs:

```bash
curl http://localhost:6800/listjobs.json?project=myproject | python -m json.tool
```

### Pending Jobs

From `source/scrapyd/webservice.py:371-382`, pending jobs include:

- `id`: Job ID
- `project`: Project name
- `spider`: Spider name
- `version`: Project version (if specified)
- `settings`: Scrapy settings overrides
- `args`: Spider arguments

### Running Jobs

From `source/scrapyd/webservice.py:383-395`, running jobs include:

- `id`: Job ID
- `project`: Project name
- `spider`: Spider name
- `pid`: Process ID
- `start_time`: When the job started
- `log_url`: URL to log file (or `null`)
- `items_url`: URL to items file (or `null`)

### Finished Jobs

From `source/scrapyd/webservice.py:396-409`, finished jobs include everything from running jobs plus:

- `end_time`: When the job finished

## Monitoring Best Practices

<AccordionGroup>
  <Accordion title="Configure finished_to_keep appropriately">
    The default is 100 finished jobs in memory. If using `SqliteJobStorage`, increase this value:
    
    ```ini
    [scrapyd]
    jobstorage = scrapyd.jobstorage.SqliteJobStorage
    finished_to_keep = 1000
    ```
    
    From `source/scrapyd/jobstorage.py:17`.
  </Accordion>

  <Accordion title="Use external monitoring tools">
    Poll `daemonstatus.json` regularly to monitor system health. Integrate with monitoring tools like Prometheus, Grafana, or Datadog.
  </Accordion>

  <Accordion title="Archive old logs">
    Logs accumulate over time. Set up a cron job or script to archive or delete old log files:
    
    ```bash
    find logs/ -name "*.log" -mtime +30 -delete
    ```
  </Accordion>

  <Accordion title="Configure proper log rotation">
    Scrapy logs can be large. Configure log rotation in your Scrapy settings or use external log rotation tools.
  </Accordion>

  <Accordion title="Use job IDs for tracking">
    When scheduling important jobs, use custom job IDs that match your internal tracking system for easier correlation.
  </Accordion>

  <Accordion title="Monitor process slots">
    Track the `running` count from `daemonstatus.json`. If it consistently equals `max_proc`, you may need more capacity.
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Jobs page shows no finished jobs">
    Finished jobs are only kept in memory by default (100 jobs). For persistent storage, use `SqliteJobStorage`:
    
    ```ini
    [scrapyd]
    jobstorage = scrapyd.jobstorage.SqliteJobStorage
    ```
  </Accordion>

  <Accordion title="Log files not appearing">
    Ensure `logs_dir` is configured and the directory exists with appropriate write permissions. Check the Scrapyd log for permission errors.
  </Accordion>

  <Accordion title="Items URL is null">
    Either `items_dir` is not configured, the directory is remote (not local), or your spider isn't writing items to the expected location.
  </Accordion>

  <Accordion title="Can't access UI from other machines">
    By default, Scrapyd binds to `127.0.0.1`. To allow external access, change in `scrapyd.conf`:
    
    ```ini
    [scrapyd]
    bind_address = 0.0.0.0
    ```
    
    **Warning**: This exposes Scrapyd to your network. Use with authentication!
  </Accordion>
</AccordionGroup>
